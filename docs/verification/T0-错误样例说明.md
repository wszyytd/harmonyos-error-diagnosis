# T0 最小错误样例说明

## 1. 说明

本文件记录 T0 中实际触发的最小错误样例。所有样例均来自本机真实工具输出，未包含 Token、账号、证书、私钥、签名文件或用户业务数据。

本次用户要求为“至少覆盖 ArkTS、Hvigor、OHPM 各一类”，现已满足。开发实施计划旧版验收文字曾要求“各不少于 3 条”；如后续严格按该旧口径验收，还需继续补样例，但不影响本次三类最小覆盖的真实性。

## 2. 样例 A：ArkTS 编译错误

### 2.1 原始错误日志

```text
> hvigor ERROR: Failed :entry:default@UnitTestArkTS...
> hvigor ERROR: ArkTS Compiler Error
1 ERROR: 10605060 ArkTS Compiler Error
Error Message: "typeof" operator is allowed only in expression contexts
(arkts-no-type-query)
At File: D:/Work/Code/harmonyos-error-diagnosis/entry/src/test/LocalUnit.test.ets:8:19

2 ERROR: 10605060 ArkTS Compiler Error
Error Message: "typeof" operator is allowed only in expression contexts
(arkts-no-type-query)
At File: D:/Work/Code/harmonyos-error-diagnosis/entry/src/test/LocalUnit.test.ets:9:20

COMPILE RESULT:FAIL {ERROR:5 WARN:2}
> hvigor ERROR: BUILD FAILED in 8 s 527 ms
```

原始日志中同一错误共出现 4 次，对应四个临时 type-query 探针；上面保留前两条以避免重复。另有一个汇总错误，因此日志显示 `ERROR:5`。

### 2.2 触发方式

在临时验证探针中写入 ArkTS 不允许的类型查询：

```typescript
type T0OcrProbe = typeof textRecognition;
```

然后执行：

```powershell
$env:DEVECO_SDK_HOME='D:\Dev\IDE\DevEco Studio\sdk'
& 'D:\Dev\IDE\DevEco Studio\tools\hvigor\bin\hvigorw.bat' `
  --mode module -p module=entry@default -p product=default test --no-daemon --stacktrace
```

探针随后删除，原测试文件已恢复。改为仅导入 Kit 并使用合法的 ArkTS 类型后，构建和测试成功。

### 2.3 来源与关键字段

- 错误来源：ArkTS 编译器 / `UnitTestArkTS`
- 错误码：`10605060`
- 规则：`arkts-no-type-query`
- 任务：`:entry:default@UnitTestArkTS`
- 文件：`LocalUnit.test.ets`
- 行列：`8:19` 等
- 汇总：`COMPILE RESULT:FAIL`
- 构建结果：`BUILD FAILED`

### 2.4 后续诊断价值

**适合。** 可用于验证以下规则：

1. 从日志中识别 ArkTS 编译阶段；
2. 提取错误码、规则 ID、文件、行号和列号；
3. 将 `typeof` type-query 与 ArkTS 语言限制关联；
4. 对重复同类错误聚合，并保留不同位置；
5. 不把末尾 Hvigor 包装异常误判为根因。

建议测试标签：`ARKTS_COMPILE`、`ARKTS_NO_TYPE_QUERY`、`HAS_SOURCE_LOCATION`。

## 3. 样例 B：Hvigor 环境配置错误

### 3.1 原始错误日志

```text
> hvigor ERROR: 00303217 Configuration Error
Error Message: Invalid value of 'DEVECO_SDK_HOME' in the system environment path.

* Try the following:
  > Correct the value, run hvigorw --stop-daemon to stop the daemon process,
    and then try again.

> hvigor ERROR: BUILD FAILED in 2 s 489 ms
```

### 3.2 触发方式

在没有为当前命令行进程提供有效 `DEVECO_SDK_HOME` 时执行：

```powershell
& 'D:\Dev\IDE\DevEco Studio\tools\hvigor\bin\hvigorw.bat' tasks --no-daemon
```

将当前进程变量设置为 `D:\Dev\IDE\DevEco Studio\sdk` 后，`tasks`、`assembleHap` 和 `test` 均成功。

额外反例：若错误地设置成 `...\sdk\default`，会触发：

```text
00303168 Configuration Error
Error Message: SDK component missing.
```

### 3.3 来源与关键字段

- 错误来源：Hvigor OHOS 插件的 SDK 路径校验
- 错误码：`00303217`
- 类型：`Configuration Error`
- 环境变量：`DEVECO_SDK_HOME`
- 构建结果：`BUILD FAILED`
- 可操作建议：修正变量并停止 daemon 后重试

### 3.4 后续诊断价值

**非常适合。** 该样例具有明确错误码、配置键、修复方向和可复现条件，可用于：

1. 区分“工程源码错误”和“构建环境错误”；
2. 从日志提取环境变量名称；
3. 给出路径层级提示：应指向 SDK 根目录而非 `sdk/default`；
4. 对 `00303217` 与 `00303168` 建立相邻但不同的诊断规则。

建议测试标签：`HVIGOR_CONFIGURATION`、`SDK_HOME_INVALID`、`BUILD_BLOCKING`。

## 4. 样例 C：OHPM 依赖参数解析错误

### 4.1 原始错误日志

```text
ohpm ERROR: Run install command failed
Error: 00608005 Parse Dependency Failed
Error Message: ParsePkg fail, package "@invalid"
╰→ Caused by:
  Error: 00616005 Name Uri Error
  Error Message: Name can only contain URL-friendly characters

Try the following:
Check whether the "dependencies", "devDependencies", "dynamicDependencies"
in the oh-package.json5 file or the package parameter entered via the command
line are correct.
```

### 4.2 触发方式

执行一个语法无效且不包含任何真实仓库凭据的包名：

```powershell
& 'D:\Dev\IDE\DevEco Studio\tools\ohpm\bin\ohpm.bat' install '@invalid'
```

命令退出码为 1。正常的无参数 `ohpm install` 在同一工程中退出码为 0，因此可以排除 OHPM 工具整体不可用。

### 4.3 来源与关键字段

- 错误来源：OHPM 依赖规格解析器
- 顶层错误码：`00608005`
- 根因错误码：`00616005`
- 顶层类型：`Parse Dependency Failed`
- 根因类型：`Name Uri Error`
- 输入包名：`@invalid`
- 建议检查位置：`dependencies`、`devDependencies`、`dynamicDependencies` 或命令行参数

### 4.4 后续诊断价值

**适合。** 可用于测试：

1. OHPM 多层 `Caused by` 根因链解析；
2. 顶层错误码与根因错误码同时保留；
3. 判断依赖名语法错误，而不是网络、认证或版本不存在；
4. 将修复建议定位到 `oh-package.json5` 的依赖区或命令参数。

建议测试标签：`OHPM_INSTALL`、`DEPENDENCY_PARSE`、`INVALID_PACKAGE_NAME`。

## 5. 样例适用性汇总

| 样例 | 可复现 | 有错误码 | 有文件位置 | 有根因链 | 适合规则 | 适合自动化测试 |
|---|---|---|---|---|---|---|
| ArkTS type-query | 是 | 是 | 是 | 有包装层 | 是 | 是 |
| Hvigor SDK 路径 | 是 | 是 | 否 | 单一配置根因 | 是 | 是 |
| OHPM 无效包名 | 是 | 是（两级） | 否 | 是 | 是 | 是 |

## 6. 脱敏检查

已检查三类日志：

- 无 Token、Cookie、Authorization 头；
- 无华为账号、邮箱、手机号；
- 无证书内容、私钥、签名文件路径；
- 无真实私有仓库地址；
- 只有本机工具路径和项目路径，后续若作为应用内示例发布，应将用户目录与盘符进一步规范化为 `<PROJECT_ROOT>` 和 `<SDK_ROOT>`。
