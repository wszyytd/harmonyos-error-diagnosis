# 功能实现与演示说明

## 演示顺序

1. 首页检查四个底部导航项是否等宽、文字基线一致、内容不被遮挡。
2. 进入“智能”，在 AI 页选择“免费端侧”，发送 HarmonyOS 报错问题。
3. 切换 Agent，查看 Planner、Skill Router 和 Verifier 步骤；可填写平台注册的 Agent ID 检测系统支持。
4. 切换 Skill，依次演示 JSON 格式化、错误分流、报告生成和隐私脱敏。
5. 切换视觉 AI，选择报错截图并提取文字；也可从首页“截图诊断”直接完成 OCR 后诊断。
6. 切换多窗，点击互动卡片或摇一摇，再创建诊断闪控子窗。
7. 切换 API，启动 `backend/server.js` 后检测连接；停止服务再次提问，验证自动降级。
8. 完成一条文本诊断，保存报告并在“记录”中查看。

| --- | --- | --- |
| 沉浸光感组件 | `uiMaterial.ImmersiveMaterial`、交互光感、局部核心卡片 | 普通玻璃背景仍可读 |

| 互动卡片 | 加速度传感器摇一摇 + 状态卡片 | 点击卡片 |
| 闪控窗/多窗 | WindowStage 应用子窗 | 页面内状态提示 |

## 隐私与安全

- 默认不联网，诊断文本仅在当前会话处理。
- OCR 使用系统选择器按次授权并优先端侧执行。
- API 模式由用户主动选择，密钥不写入仓库。
- 演示后端不存储请求，不包含账号、口令或第三方付费模型密钥。
