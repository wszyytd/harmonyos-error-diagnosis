# T17 浅色沉浸光感设计

## 设计结论

依据华为 API 26 沉浸光感指南，浅色主题不再使用蓝色常驻霓虹模拟玻璃，而是使用系统 `ImmersiveMaterial` 表达内容相关的毛玻璃、材质阴影和按压点光源。深色主题保持提交 `cdc2c09` 的蓝色玻璃样式不变。

## 浅色主题

- 页面与导航背景使用冷中性灰 `#ECEEF2`，为白色玻璃提供可感知的明暗参照。
- 普通卡片使用近白色 `#F8F9FB`，不添加发光效果。
- 三个既有焦点使用半透明白色表面 `#66FFFFFF`、高亮白边 `#E6FFFFFF` 和柔和白色光晕 `#99FFFFFF`。
- 焦点组件启用 `uiMaterial.ImmersiveMaterial`：`ImmersiveStyle.THIN`、`interactive: true`、默认白色 `lightEffect`、系统材质阴影。
- 首页仍只有文本诊断入口使用该材质；诊断页仍只有“开始诊断”按钮；报告页仍只有结论头部。

## 深色主题与降级

- 深色主题的背景、表面、边框和蓝色光晕值全部保持不变。
- 深色主题不启用本轮新增的系统材质，以避免改变已提交效果。
- 系统材质在模拟器或设备上不可见时，半透明白色、白边和静态白色光晕继续提供可辨认的焦点。
- 不引入 HDS Navigation/Tabs 迁移，不改变页面结构、业务逻辑、路由或测试边界。

## 验证标准

- 本机 API 26 SDK 能编译 `systemMaterial`、`uiMaterial.ImmersiveMaterial` 和 `ImmersiveStyle.THIN`。
- 浅色主题背景明显为灰色，焦点呈白色模糊玻璃而非蓝色霓虹。
- 深色主题视觉令牌与 `cdc2c09` 一致。
- Local Test、ArkTS Linter、Debug HAP、ohosTest HAP 和 `git diff --check` 通过。
- 实际材质、触点光源和模糊程度由用户在 API 26 模拟器上验收。

## 2026-07-14 首页可读性修订

- “截图诊断”和“文本诊断”在浅色主题下使用相同的白色沉浸玻璃入口样式。
- 首页其余操作按钮使用浅蓝半透明玻璃，深蓝文字确保可读性，不再叠加默认蓝底与蓝字。
- 浅蓝按钮使用独立语义色，不改变诊断焦点玻璃和深色主题。
- 修订只涉及首页及首页空状态按钮，不改变导航、诊断和历史记录行为。
