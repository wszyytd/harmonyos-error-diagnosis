# T17 强光玻璃焦点设计

## 目标与范围

采用用户选定的 C「强烈霓虹」方案，使 API 26 模拟器中的玻璃焦点无需仔细观察即可辨认。改动只覆盖首页文本诊断入口、诊断页“开始诊断”按钮、报告页错误类型与结论头部；不改变诊断流程、路由、数据模型、普通内容卡片或 UiTest。

## 视觉结构

- 焦点表面：使用透明度明显低于当前 `#CC...` 的蓝紫色半透明表面，保留 `BlurStyle.Thin`。
- 第一层光感：边框由 1vp 提升到 2vp，使用高亮冷色，形成清晰玻璃轮廓。
- 第二层光感：增加无位移或轻微下移的蓝色外发光阴影；亮色与深色主题分别使用独立语义色，避免硬编码到页面。
- 层级约束：首页只有文本诊断入口发光；诊断页只有“开始诊断”按钮发光；报告页只有结论头部发光。其他卡片继续使用普通实色表面。
- 动效约束：保留现有按下缩放 `1 → 0.98` 与松开单次回弹，不增加呼吸、闪烁或其他循环动画。

## 工程改动

1. 在 `ThemePalette` 中补充外发光语义色，并增强亮色、深色主题的 `focusSurface`、`focusBorder` 配色。
2. 在 `HomePage` 的焦点入口、`DiagnosisPage` 的开始按钮和 `ReportPage` 的结论头部应用统一的 2vp 边框与外发光阴影。
3. 不抽取新组件，不改动三个页面的业务事件和状态管理，避免扩大当前 T17 半成品的修改范围。

## 降级与可访问性

- 模糊能力不可用时，半透明色、2vp 高亮边框与静态阴影仍能表达焦点。
- 减少动态模式下不依赖新增动画，因此不需要额外分支。
- 阴影不承载文字信息；文本对比度、按钮可点击区域和无障碍标签保持现状。

## 验证标准

- 模拟器中无需按压即可明显看到三处指定焦点的蓝紫色光晕。
- 普通卡片不出现相同光晕，页面仍只有一个主视觉焦点。
- 深色和亮色主题下文字清晰，1.5 倍字体不影响滚动与操作。
- Local Test、ArkTS Linter、Debug HAP、ohosTest HAP 和 `git diff --check` 不因视觉修改失败。
- 不执行或调试带滚动手势的 UiTest。
