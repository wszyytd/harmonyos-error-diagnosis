# 开发报错诊断助手——项目使用与开发手册

> 适用版本：本地 Git `develop` 分支，2026-07-16 完整演示版
>
> 目标平台：HarmonyOS API 26，ArkTS，ArkUI Stage 模型
>
> 项目目录：`D:\harmonyos-error-diagnosis`

## 1. 项目简介

开发报错诊断助手是一款面向程序员、计算机专业学生和课程设计团队的 HarmonyOS 原生效率应用。应用可以整理 ArkTS、Hvigor、OHPM、JSON、HTTP、权限和网络等常见开发错误，生成结构化诊断报告，并在本地保存历史记录。

项目采用“本地优先、联网增强”的设计：基础诊断、报告、记录、工具和设置在断网状态下仍可使用；用户主动切换在线 API 并提交文本时，才会调用配置的模型服务。

## 2. 功能总览

| 模块 | 主要功能 | 是否依赖网络 |
| --- | --- | --- |
| 首页 | 截图诊断、文本诊断、最近记录、动态光感、底部导航 | 否 |
| 本地诊断 | 输入校验、规则匹配、错误分类、原因与步骤生成 | 否 |
| 报告 | 结构化结论、证据、建议、保存、重新诊断 | 否 |
| 记录 | 本地历史、报告详情、清空记录 | 否 |
| AI 对话 | 真实 OpenAI 兼容模型 API、离线规则降级 | 在线模型需要 API Key |
| Agent | 本地 Planner/Skill Router/Verifier、系统 Agent 能力检测 | 系统 Agent 视平台而定 |
| Skill | JSON Formatter、Error Triage、Report Writer、Privacy Redactor | 否 |
| 视觉 AI | 系统图片选择器、Core Vision Kit OCR、文本诊断衔接 | OCR 视设备能力而定 |
| API 工具 | 后端健康检查、JSON 格式化、HTTP 状态码查询 | 健康检查需要 |
| 多窗 | 互动诊断卡片、可打开/关闭的诊断闪控子窗 | 否 |
| 设置 | 主题、字号、光效、诊断偏好、隐私、AI 服务商、超时、记录容量 | 否 |

## 3. 开发环境

本项目当前验证环境如下：

- DevEco Studio：26.0.0.461
- HarmonyOS SDK：26.0.0.23，API 26
- Hvigor：6.26.1
- Node.js：用于零依赖演示后端
- 模拟器：Pura 90 Pro，`127.0.0.1:5555`

本机实际工具路径：

```text
D:\downloading\DevEco Studio
D:\downloading\DevEco Studio\sdk
D:\downloading\DevEco Studio\sdk\default\openharmony\toolchains\hdc.exe
```

如果在其他电脑运行，请把命令中的 DevEco Studio 路径替换为本机路径。

## 4. 打开项目

### 4.1 DevEco Studio

1. 启动 DevEco Studio。
2. 选择“打开项目”。
3. 选择 `D:\harmonyos-error-diagnosis`，不要只打开 `entry` 子目录。
4. 等待 OHPM 同步和索引完成。
5. 在设备列表选择已启动的 Pura 90 Pro 模拟器。
6. 选择 `entry`，点击运行。

### 4.2 VS Code

在 VS Code 中打开 `D:\harmonyos-error-diagnosis`。项目使用本地 Git 管理，日常开发分支为 `develop`。

```powershell
cd D:\harmonyos-error-diagnosis
git branch --show-current
git status
```

正常情况下当前分支应显示 `develop`。

## 5. 启动模拟器与应用

1. 在 DevEco Studio 右上角打开“设备管理器”。
2. 启动 Pura 90 Pro 模拟器。
3. 等待模拟器进入桌面。
4. 在 DevEco Studio 选择该设备并运行 `entry`。

也可以使用命令安装已经构建的 HAP：

```powershell
$hdc = 'D:\downloading\DevEco Studio\sdk\default\openharmony\toolchains\hdc.exe'
& $hdc list targets
& $hdc -t 127.0.0.1:5555 install -r `
  'D:\harmonyos-error-diagnosis\entry\build\default\outputs\default\entry-default-unsigned.hap'
& $hdc -t 127.0.0.1:5555 shell aa start `
  -a EntryAbility -b com.example.errordiagnosis
```

## 6. 后端启动与 API 连通

项目后端位于 `backend/`，仅使用 Node.js 标准库，没有第三方运行依赖。

推荐在模拟器启动后执行：

```powershell
cd D:\harmonyos-error-diagnosis
.\backend\start-emulator-api.ps1
```

脚本会完成三件事：

1. 启动 `backend/server.js`；
2. 建立 `hdc rport tcp:8787 tcp:8787` 端口转发；
3. 请求 `/health` 确认服务可用。

成功时会显示：

```text
API ready: ok, model=free-rule-model-v1, device=127.0.0.1:5555
```

也可以手动启动：

```powershell
cd D:\harmonyos-error-diagnosis\backend
npm test
npm start
```

应用内本地 API 地址为：

```text
http://127.0.0.1:8787
```

该地址依赖 HDC 反向端口转发。未建立转发时，应用会提示 API 不可达，并保留本地免费模型作为降级方案。

## 7. 使用真实在线模型

应用支持 OpenAI Chat Completions 兼容接口。进入“智能”页面后：

1. 选择“在线 API”。
2. 输入 API Key。
3. 输入模型名称。
4. 填写模型服务地址（默认提供 Groq OpenAI 兼容入口）。
5. 点击检测连接，确认密钥、模型 ID 和网络可用。
6. 返回 AI 或 Agent 标签，输入任务并发送。

Groq 示例：

```text
API 地址：https://api.groq.com/openai/v1
模型名称：openai/gpt-oss-20b
API Key：用户自己在服务商控制台申请
```

同样可以填写其他 OpenAI 兼容服务，例如 SiliconFlow 或自建兼容服务，并使用服务商实际提供的模型 ID。

安全说明：

- API Key 只保存在当前应用运行内存中；
- 不写入 Preferences、数据库、Git 或项目文档；
- 不要把真实 Key 截图、提交或发给他人；
- 只有用户点击发送后，输入框中的文本才会发给在线服务。

## 8. 离线规则诊断

`DevAssist Mini` 是项目内置的零费用离线规则诊断引擎。它由规则库、意图识别、报告模板和 Skill 编排组成；界面明确标记为“非生成式 AI”，不会把固定模板伪装成通用大语言模型。

支持的主要类别包括：

- OHPM 包不存在、依赖版本和 registry 问题；
- ArkTS 类型和编译错误；
- 权限声明与运行时授权问题；
- JSON 语法和字段问题；
- 网络超时和通用排查流程。

当在线 API 不可达时，应用会自动回退到该模型，保证课程现场仍能演示。

## 9. 各页面使用说明

### 9.1 首页

- “截图诊断”：进入图片选择和 OCR 流程；
- “文本诊断”：直接输入或粘贴错误日志；
- “打开验证工具”：进入智能工具中心；
- “查看全部记录”：打开本地历史；
- 底部 HM/AI/RC/ST 分别对应首页、智能、记录和设置。

首页背景包含缓慢移动的蓝、青和暖色动态光团。设置中选择“减少光效”会降低动画强度，选择“关闭光效”后功能仍可正常使用。

### 9.2 文本诊断

1. 输入错误日志，或点击示例文本。
2. 点击开始诊断。
3. 查看错误类型、摘要、证据、可能原因和解决步骤。
4. 主动保存报告后，记录才会写入本地数据库。

### 9.3 截图与视觉 AI

1. 点击选择报错截图。
2. 使用系统图片选择器授权单张图片。
3. 支持时通过 Core Vision Kit OCR 提取文字。
4. 将识别文字继续交给本地诊断或 AI。

截图页内置 4 张典型报错图片。首次点击“保存至图库”，在系统安全授权弹窗选择“允许”，测试图会写入模拟器相册；随后点击“选择截图并识别文字”即可完成选择与 OCR 演示。模拟器不支持视觉能力时会显示明确提示，文本诊断仍然可用。

### 9.4 Agent

本地 Agent 使用以下流程：

```text
Planner → Skill Router → 执行步骤 → Verifier
```

在线 API 模式下，Agent 会把任务发送给配置的真实模型，并返回任务计划。系统 Agent Kit 需要平台注册的 Agent ID 和设备能力；模拟器不支持时不会影响本地 Agent。

### 9.5 Skill

- JSON Formatter：解析并格式化 JSON；
- Error Triage：根据输入快速分类错误；
- Report Writer：生成“现象—分析—步骤—验证”报告；
- Privacy Redactor：隐藏本地路径、邮箱、Token、API Key 和密码形式的文本。

### 9.6 多窗和互动卡片

- 互动卡片可点击或通过传感器改变状态，模拟器可直接点击“直接展示互动卡片”；
- “打开独立诊断窗口”创建真实应用子窗；
- 子窗显示诊断进度，支持暂停、恢复、继续；
- 可从主页面或子窗内部关闭；
- 关闭后窗口被销毁，再次打开不会叠加旧窗口。

### 9.7 记录与设置

诊断记录保存在设备本地 relationalStore 中。主题、字号和光效保存在 Preferences 中。设置页还提供自动保存、剪贴板检测、隐私脱敏、触觉反馈、通知、AI 服务商、请求超时和记录容量等演示选项。隐私页面说明数据边界和联网行为。

### 9.8 API/工具页

- “真实模型 API 配置”用于检查模型服务连接；真实生成式回答在 AI/Agent 页触发。
- JSON 格式化和 HTTP 状态码速查是本地确定性工具，不调用模型，也不会根据文本生成答案。
- 如果任何输入都得到相同回答，请确认选择的是“在线 API”，并已经填写有效 API Key 与模型 ID；“离线规则诊断”只处理有限的开发错误类型。

## 10. 后端接口

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| GET | `/health` | 健康检查 |
| GET | `/api/models` | 返回本地免费模型信息 |
| POST | `/api/chat` | 本地 API 聊天诊断 |
| POST | `/api/agent/run` | Agent 规划与 Skill 流程 |
| POST | `/api/vision/analyze` | 分析已脱敏的 OCR 文本 |

聊天请求示例：

```json
{
  "prompt": "HarmonyOS 工程出现 ohpm package not found"
}
```

后端返回回答、服务提供者、分类、处理步骤和 requestId。空输入返回 `422`，错误路由返回 `404`，无效 JSON 返回 `400`。

## 14. 常见问题

### API 显示不可达

1. 确认 Node.js 后端正在运行；
2. 执行 `backend\start-emulator-api.ps1`；
3. 确认模拟器目标为 `127.0.0.1:5555`；
4. 确认应用内地址为 `http://127.0.0.1:8787`；
5. 在线模型还需要检查 API Key、模型 ID 和网络。

### 出现 `Invalid value of DEVECO_SDK_HOME`

重新设置：

```powershell
$env:DEVECO_SDK_HOME='D:\downloading\DevEco Studio\sdk'
```

### 模拟器没有绿色运行按钮

确认设备管理器中的模拟器已启动，并等待设备列表出现 Pura 90 Pro，再选择 `entry` 运行配置。

### Agent 或 OCR 提示设备不支持

系统 Agent、Skill 分发和 Core Vision Kit 能力与设备、SDK、开发者认证及平台注册状态有关。应用会保留本地 Agent、Skill、文本诊断和点击互动等降级功能。

### 闪控窗一直存在

可在多窗页面点击“关闭闪控窗”，也可在子窗内部点击关闭。应用退出时也会自动释放子窗。

## 16. 能力边界

本项目已经完成可编译、可运行、可交互的应用主体和后端。系统级小艺 Agent、系统 Skill 分发、系统实况窗等能力仍受华为平台开通、实名认证、服务配置和目标设备支持限制。项目不会把“SDK 中存在接口”描述成“所有设备已经开通”，不支持时会显示真实状态并使用应用内实现继续演示。
